# Food-Delivery-Business-Analytics-using-SQL
SQL-based Food Delivery Analytics project using SQL Server to analyze revenue, customer behavior, delivery performance, delays, traffic impact, and operational insights.

## Project Overview

This project analyzes a food delivery business dataset using SQL Server to extract insights related to revenue, customer behavior, delivery performance, and operational efficiency.

The goal is to answer real-world business questions using SQL-based data analysis.

---

## Dataset Information

- Source: Kaggle Food Delivery Dataset  
- Database: FoodDeliveryDB  
- Table: Orders  
- Tool: Microsoft SQL Server Management Studio  

Dataset includes:
- Order details (value, final amount, discounts, tips)
- Customer data (age, loyalty score, premium status)
- Delivery metrics (time, distance, experience)
- Operational flags (delay, cancellation, refund)
- Environmental factors (traffic, weather)
- Time attributes (hour, month, city tier)

---

## Business Questions Answered

### Revenue Analysis
- Total revenue generated
- Average order value
- Monthly revenue trends
- Promo code impact
- Total tip amount
- High-value orders

### Customer Analysis
- Average customer age
- Premium vs non-premium customers
- Spending comparison
- Loyalty score analysis

### Delivery Performance Analysis
- Average delivery time
- Average delivery distance
- Delayed delivery count
- Impact of traffic on delivery time
- Impact of weather on delivery time
- Delivery partner experience effect

### Operational Analysis
- Peak order hours
- Cancellation count
- Refund count
- City-wise revenue contribution
- Order value segmentation

---

## SQL Techniques Used

- Aggregate functions (SUM, AVG, COUNT)
- GROUP BY analysis
- CASE WHEN logic
- WHERE filtering
- ORDER BY sorting
- Time-based grouping (hour, month)
- Business logic creation using SQL

---

## Key Insights

- Orders peak during specific hours
- Premium customers spend more on average
- Traffic and weather affect delivery time
- Experienced delivery partners perform better
- Revenue varies across city tiers
- Delays and refunds show inefficiencies
- Promo codes influence revenue behavior
